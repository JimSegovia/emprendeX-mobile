# Documentacion del proyecto

Este directorio concentra la documentacion externa del proyecto.

## Indice

1. `architecture/overview.md`
   Mapa tecnico general del repositorio, responsabilidades por carpeta y puntos de mantenimiento.

2. `architecture/auth-onboarding-navigation.md`
   Explica como se resuelve el acceso de un usuario desde login hasta la navegacion interna.

3. `domain/modules.md`
   Resume los modulos funcionales, sus pantallas principales y sus dependencias de dominio.

4. `maintenance/documentation-guidelines.md`
   Define como mantener consistente la documentacion del proyecto a futuro.

## Cuando actualizar estos documentos

1. Cuando cambie el flujo `auth -> onboarding -> drawer/tabs`.
2. Cuando se agregue o elimine un modulo funcional.
3. Cuando cambie la responsabilidad de una carpeta o capa compartida.
4. Cuando una decision tecnica impacte como otros desarrolladores extienden la app.
